# GD Math Fixes and Suggestions

## Onboarding and Assessment Improvements
- **Add Demo Levels**: Include example/demo levels before assessment levels for every skill age to familiarize users with gameplay.
- **Welcome Message Timing**: Display the onboarding welcome message screen when users complete the profile creation process or click the assessment button for the first time.

## UI/UX Enhancements
- **Profile List Screen Animation**: When a first-time user opens the app, completes onboarding, and saves their profile, the profile list screen appears with a play button that zooms in/out. Increase the zooming value for better visibility to help users notice the button.
- **Home Screen Assessment Button**: Once first-time users land on the home page, make the assessment button(brain button) hover/animate to draw attention.
- **Button Design Consistency**: Use protruding 3D-style buttons with text labels throughout the app for better visual hierarchy and usability.

## Age Gate and Subscription Issues
- **Age Gate Barrier**: Analytics show that many users drop off at age gates wherever they are placed, acting as significant barriers. Many users reach the age gate but fail to proceed to the player report, which is a critical concern requiring fixes.
- **Subscription Screen Similarity**: Similar drop-off issues exist for the subscription screen, where users hit barriers and don't convert.
- **Screen Orientation Change**: When hitting the age gate, the screen orientation forcibly changes, which is not user-friendly and contributes to poor experience.
- **Age Gate Solutions**: Either redesign the age gate to be more user-friendly or remove it entirely before the player report and subscription screens to reduce friction.
