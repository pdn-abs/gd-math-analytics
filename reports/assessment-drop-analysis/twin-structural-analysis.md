## Three Root Reasons ‚Äî Why Twins Escape the Problematic List

### Pattern A ‚Äî "Not problematic" only by exclusion rule (Pairs 1, 6, 8)

The most important finding: **these twins ARE problematic ‚Äî they're just invisible to the analysis** because they are the first level in their SkillAge and are excluded by the non-first-level scope of this report.

| Twin | SA | Rank | Actual Drop Rate | Would it be in the problematic list? |
|---|:---:|:---:|:---:|---|
| `numbersFrom500To999AL` | 8 | 1/5 | **46.7%** | Yes ‚Äî Tier 1 if not excluded |
| `simpleIdentificationVegetablesSet2AL` | 2 | 1/4 | **42.9%** | Yes ‚Äî Tier 1 if not excluded |
| `stackingDifferentPairsOfSameValue202AL` | 6 | 1/6 | **47.4%** | Yes ‚Äî Tier 1 if not excluded |

These twins drop from **onboarding friction** (users encountering the assessment UI for their SkillAge for the first time). The problematic levels they're paired with have a different problem ‚Äî content mechanics or engagement failure.

**Implication:** For pairs 1, 6, and 8, you do **not** have a "good twin vs bad level" situation. You have **two levels both needing attention** ‚Äî one from onboarding friction, one from content or engagement failure. The twin is not a healthy baseline.

---

### Pattern B ‚Äî Older age group or late sequence position (Pairs 2, 4, 9)

> **Important correction:** Each user is shown assessment levels for their own SkillAge only, determined by DOB during onboarding. An SA8 user has never played SA5 levels ‚Äî they are a different (older) child. Cross-SA comparisons are about **developmental maturity**, not accumulated game experience.

Two distinct sub-patterns apply here:
- **Cross-SA pairs (2, 9):** The twin's users are older children (higher SA = higher DOB-based SkillAge). They have stronger cognitive development, better working memory, and more advanced subject-matter knowledge for their age group.
- **Same-SA pair (4):** Within a single SkillAge, users who reach a later level have already completed earlier levels in that same SA. This within-SA survivorship is real ‚Äî users who quit earlier in the sequence are absent from later levels.

#### Pair 2: `orderUpToSevenNumber2AL` (SA5, rank 2/5) vs `descendingOrdering1to999by50AL` (SA8, rank 2/5)

Same relative rank, 3 SAs apart ‚Äî but these are entirely different age groups. SA5 and SA8 users have never shared a level; they each see only their own SA's content.

- **SA8 users are older children** with stronger number sense and greater familiarity with larger numbers (up to 999). Descending ordering of multiples of 50 is genuinely cognitively challenging for their age, which creates engagement ‚Äî they have a reason to focus and act.
- **SA5 users see 1‚Äì7 ascending ordering**, which is trivially obvious for a child at the SA5 developmental level. The correct order is instantly recognizable, removing any cognitive challenge. There is nothing to solve, so there is no motivation to place the tiles. Users disengage before acting (drop WM = 1.09 ‚Äî fewer than 2 tile placements before quitting).

The twin's lower drop rate is explained by **content difficulty relative to user developmental level**, not by game experience.

#### Pair 4: `matrixMultiplicationRandomNumbers3AL` (SA10, rank 3/4) vs `givenSmartChartScale6AL` (SA10, rank 4/4)

Same SkillAge, one position apart. `givenSmartChartScale6AL` is the very last SA10 level. Only users who cleared the first three SA10 levels (including `matrixMultiplicationRandomNumbers3AL` itself) reach it. This is the most battle-hardened SA10 population ‚Äî they have already proved they can tolerate difficult SA10 content. Matrix at rank 3/4 still encounters a broader population that includes users who haven't been filtered by 4 hard levels.

#### Pair 9: `stackColorTilesSet1And22AL` (SA2, rank 3/4) vs `sortRedGreenYellowWhiteBlackGray2AL` (SA3, rank 6/7)

Two separate factors both favor the twin here:

1. **Within-SA3 sequence position (survivorship):** The twin is rank 6 of 7 within SA3. SA3 users who reach it have already completed 5 prior SA3 assessment levels. Users who tend to quit early have done so on earlier SA3 levels ‚Äî the population reaching rank 6 is self-selected for persistence within that age group.
2. **Developmental age:** SA3 users are slightly older than SA2 users. This alone is a minor factor, but it compounds with sequence position.

Importantly, SA2 and SA3 are different age groups entirely ‚Äî SA2 rank 3 users and SA3 rank 6 users have never played the same levels. The cross-SA comparison here is developmental, not experiential.

### Pattern C ‚Äî Simpler task design (Pairs 3, 7)

The twin variant has lower cognitive demand for the same underlying concept. SA position provides no advantage here ‚Äî in pair 3, the twin is actually at a *lower* SA than the problematic level.

#### Pair 3: `stackMultiStickInMultiSlot1To52AL` (SA4, rank 2/4) vs `stackMultiStick1To52AL` (SA3, rank 2/7)

- **Twin (SA3):** One labeled count slot per board. Count the sticks, drag to the slot. One operation, one unknown. Drop rate: **13.6%**.
- **Problematic (SA4):** Three simultaneous slots (x, a, and b where a+b=x). The user must satisfy an interdependency across all three values at once. Drop rate: **39.7%**. Completion time: **162s vs 47s**.

No amount of user experience at SA4 overcomes a layout that demands tracking three mutually constrained values simultaneously. This exceeds the working memory capacity of the target age group. It is a **task architecture problem**, not an experience or SA problem.

#### Pair 7: `writeLevelSimpleLine2AL` (SA3, rank 7/7 [LAST]) vs `writeLevel1To52AL` (SA4, rank 4/4 [LAST])

- **Twin (SA4):** Traces numeral shapes 1‚Äì5. Each numeral is a culturally familiar symbol with an obvious success moment ‚Äî the user knows what "correct" looks like.
- **Problematic (SA3):** Traces abstract lines. No recognizable reference shape. The user cannot tell whether their trace was good or not, so there is no feedback loop to reinforce continuation. Mean drop time: **26s** vs **51s** for the twin.

Tracing a recognizable numeral is self-reinforcing. Tracing an abstract line is ambiguous. The problem is feedback clarity, not difficulty.

### Pattern D ‚Äî The "twin" is not actually healthy (Pair 5)

#### Pair 5: `fractionMultiplication6AL` (SA10, rank 2/4) vs `multiplication3TableAL` (SA8, rank 4/5)

`multiplication3TableAL` drops at **50%** ‚Äî higher than the "problematic" level's 24.5%. The twin is not a healthy comparator.

Why does multiplication3Table not appear in the problematic list?
- It is at SA8, Seq265 (4th of 5 levels), where user counts are small ‚Äî likely below the ‚â•20 started volume threshold used to qualify levels for the report.
- `fractionMultiplication6AL` is labeled problematic not for drop rate but for its **done WM/user of 12.93** (completers need ~13 wrong attempts on average) and **done success rate of only 54.4%**.

These are different failure modes:
- fractionMult: **fewer droppers, but completers suffer** (content execution difficulty)
- multiplication3Table: **more droppers, but from passive disengagement** (drop WM = 1.00 ‚Äî users barely try before leaving)

This is not a healthy-vs-problematic pairing. Both need attention through different interventions.
